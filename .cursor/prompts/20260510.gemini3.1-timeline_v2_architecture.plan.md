# Timeline v2 Architecture

Here is the straightforward, KISS-compliant architecture for the timeline module. We don't need exotic tree structures or new dependencies—your existing Bit-Sliced Index (BSI) with Roaring Bitmaps can handle this if we adapt it properly. 

### 1. Ditch the complex tree structures
Don't bother with Interval Trees, Quadtrees, or 1D Geohashes. They add unnecessary complexity and code bloat. BSI inherently handles range queries efficiently by scanning bit slices. We stick to BSI.

### 2. Dual-BSI for Intervals
To support intervals (`<start, end>`), we simply maintain **two** BSIs per timeline: one for `start` times and one for `end` times. 
A point-in-time event is just an interval where `start == end`. 

To find overlapping events (the "zeitgeist" use case) for a queried range `[Q_start, Q_end]`, the logic is trivially:
```javascript
const overlapping = RoaringBitmap32.and(
    await timeline.start.query('<=', Q_end),
    await timeline.end.query('>=', Q_start)
);
```

### 3. Upgrade BSI to 64-bit (BigInt)
Currently, `BitSlicedIndex.js` uses 32-bit bitwise operators (e.g., `value >>> i`). This caps out quickly. We will upgrade the BSI implementation to use JavaScript `BigInt` with `bitDepth = 64`. 
- 64 bits gives us an address space large enough to track the age of the universe in seconds, or +/- millions of years in milliseconds.
- We map `-∞` to `0n` (offsetting the epoch) so the BSI only deals with non-negative BigInts. No need for complex hierarchical adaptive bucketing.

### 4. Dynamic Composability
We rip out the hardcoded `created`, `updated`, and `deleted` BSIs in `[src/indexes/inverted/Timestamp.js](src/indexes/inverted/Timestamp.js)`. Instead, `TimestampIndex` will lazily instantiate `start` and `end` BSIs based on arbitrary timeline names (`wikipedia`, `family`, `system_actions`).

### 5. Aggressive Code Deletion
We will nuke all the `date-fns` garbage (`isToday`, `startOfYesterday`, etc.) from `Timestamp.js`. The timeline index should be dumb and only deal with 64-bit integers. The application layer or a thin wrapper can handle calendar math, keeping the core indexing module extremely slim.

### Summary of Changes

- **`[src/indexes/bitmaps/lib/BitSlicedIndex.js](src/indexes/bitmaps/lib/BitSlicedIndex.js)`**: Change bitwise operators (`>>>`, `&`) to use `BigInt` logic. Allow configurable `bitDepth` up to 64.
- **`[src/indexes/inverted/Timestamp.js](src/indexes/inverted/Timestamp.js)`**: 
  - Remove all `date-fns` imports.
  - Remove `VALID_ACTIONS`.
  - Add a dynamic timeline registry (e.g., `getTimeline(name)` returns `{ start: BSI, end: BSI }`).
  - Update `insert` to accept `(timelineName, id, startInt, endInt)`.
  - Update queries to handle overlapping logic using BigInts.
